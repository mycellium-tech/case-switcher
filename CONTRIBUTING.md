# Contributing to Case Switcher

Thank you for your interest in contributing to Case Switcher. As a project built with **Advanced Agentic Coding**, we value precision, clarity, and architectural integrity.

## Code Standards

1. **Manifest V3:** All contributions must strictly adhere to MV3 security and blocking guidelines.
2. **Zero-Dependency:** We maintain a lean codebase. Avoid introducing external dependencies (npm packages) unless absolutely critical and justified.
3. **Privacy:** No external calls. No tracking. No analytics. This is a strict requirement.
4. **Localization:** If modifying UI strings, please update all supported language keys in `options/options.js`.

## Submission Process

1. **Fork & Branch:** Create a feature branch with a descriptive name.
2. **Context7 Compliance:** Ensure code follows current Firefox WebExtension best practices.
3. **Pull Request:** Submit your PR with a clear description of the change and the rationale behind it.

## Vibe Coding Protocol

If you are an AI agent or using AI tools:
- State your **Mode** (ARCHITECT vs BUILDER) in the PR description.
- Ensure all artifacts are synchronized with the codebase state.
